# Self Improvement Loop

Use this loop to let the AI team improve itself safely.

## Cadence

- Daily: micro improvements (prompts, templates, routing tweaks).
- Weekly: evaluate metrics and commit one structural upgrade.
- Monthly: remove stale rules and simplify.

## Guardrails

1. Every self-change must map to a measurable metric.
2. Keep one improvement change-set per commit.
3. Run at least one real task after change before keeping it.
4. Revert quickly if quality or cost regresses.

## Improvement Backlog Types

- Prompt quality
- Task template quality
- Workflow throughput
- Cost efficiency
- Defect escape reduction

## Definition Of Better

- Lower average lead time
- Lower cost per completed task
- Lower rollback/defect rate
- Higher first-pass QA rate
